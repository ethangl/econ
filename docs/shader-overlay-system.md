# Shader-Based Overlay System

The map uses a GPU-driven overlay system for borders, map modes, and terrain rendering.

## Key Files

- `Assets/Shaders/MapOverlayFlat.shader` - flat political/market/religion rendering style
- `Assets/Shaders/MapOverlayBiome.shader` - biome terrain + water rendering style
- `Assets/Shaders/MapOverlay.Common.cginc` - sampling helpers, hash functions, value noise, FBM
- `Assets/Shaders/MapOverlay.Composite.cginc` - terrain computation, volumetric water, slope relief shading
- `Assets/Shaders/MapOverlay.ResolveModes.cginc` - per-mode border application (political, market, religion)
- `Assets/Scripts/Renderer/MapOverlayManager.cs` - Data texture generation, spatial grid, border distance fields
- `EconSim.Core/Rendering/PoliticalPalette.cs` - Realm color generation

Shader include hierarchy:

```
MapOverlayFlat.shader / MapOverlayBiome.shader
├── MapOverlay.Common.cginc
├── MapOverlay.Composite.cginc
└── MapOverlay.ResolveModes.cginc  (Flat only — Biome has no mode-based borders)
```

## Data Textures

**PoliticalIds** (RGBAFloat, point-filtered):

- R: RealmId / 65535
- G: ProvinceId / 65535
- B: CountyId / 65535
- A: reserved

**GeographyBase** (RGBAFloat, point-filtered):

- R: BiomeId / 65535
- G: SoilId / 65535
- B: reserved
- A: WaterFlag (0 = land, 1 = water)

**Vegetation** (RGBAFloat, point-filtered):

- R: VegetationTypeId / 65535
- G: VegetationDensity (0-1)
- B: Precipitation (0-1, normalized)
- A: reserved

**Heightmap** (RFloat, bilinear): Smoothed terrain elevation, 0-1 normalized.

**River mask** (RG16): R = distance from river (normalized), G = normalized flux (for variable width).

**Colormap** (64×64): Elevation × moisture → terrain color lookup (mapgen4-style).

## Palette Textures (256×1)

- Realm palette: colors from PoliticalPalette (even hue distribution)
- Market palette: derived from hub realm colors
- Biome palette: direct biome colors

## Color Derivation (shader-based cascade)

- Political mode: realm color from palette
- Province mode: derived from realm color + hash(provinceId) HSV variance (±0.07 S/V)
- County mode: derived from province color + hash(countyId) HSV variance (±0.07 S/V)
- Realm HSV: S base 0.52 [0.38, 0.68], V base 0.70 [0.58, 0.85]
- Derived HSV clamp: S [0.15, 0.95], V [0.25, 0.95] (allows ± variance around parent)

## Market Zone Mapping

- Separate `_CellToMarketTex` (16384×1, RFloat)
- Maps CellId → MarketId via CountyToMarket lookup
- Updated dynamically when economy state changes

## Border Rendering (distance-field based)

All borders use signed distance field (SDF) textures generated by chamfer distance transforms. The shader uses `smoothstep()` + `fwidth()` for screen-space anti-aliased rendering.

### Border Hierarchy

**Political:** Realm (always visible) → Province (display level ≥ 1.5) → County (display level ≥ 2.5)

**Religious:** Archdiocese (always visible) → Diocese (display level ≥ 1.5) → Parish (display level ≥ 2.5)

**Market:** Single-tier market zone borders

### Border Distance Textures (R8, bilinear)

- `_RealmBorderDistTex` - distance to nearest realm boundary
- `_ProvinceBorderDistTex` - distance to province boundary
- `_CountyBorderDistTex` - distance to county boundary
- `_MarketBorderDistTex` - distance to market zone boundary
- `_ArchdioceseBorderDistTex` - distance to archdiocese boundary
- `_DioceseBorderDistTex` - distance to diocese boundary
- `_ParishBorderDistTex` - distance to parish boundary

Realm/archdiocese boundaries include coasts, lakes, and rivers (water neighbors count as edges). Province/county/diocese/parish boundaries are interior-only (land-to-land transitions).

### Border Resolution Scale

Border distance textures can be generated at higher resolution than the main spatial grid for smoother edges:

- `borderResolutionScale` (1.0–3.0, default 1.5) controls the multiplier
- `borderWidth = gridWidth × borderResolutionScale`
- `_BorderTexelScale` shader uniform scales border width thresholds to compensate
- The upsampled border grid is nearest-neighbor from the main spatial grid; the chamfer transform smooths staircase artifacts into continuous gradients

### Shader Border Uniforms

- `_EdgeWidth` (default 6) - edge band width in texels
- `_EdgeDarkening` (default 0.15) - edge band darkening strength
- `_RealmBorderWidth`, `_ProvinceBorderWidth`, `_CountyBorderWidth` - border line widths
- `_RealmBorderDarkening`, `_ProvinceBorderDarkening`, `_CountyBorderDarkening` - per-tier darkening
- `_MarketBorderWidth`, `_MarketBorderDarkening` - market zone border controls
- `_BorderTexelScale` - multiplier applied to all width thresholds (matches border resolution scale)

### Display Level System

Per-pixel display level is packed into the `_ModeColorResolve` alpha channel. It controls which border tiers are visible, enabling CK3-style expand/collapse of realms, provinces, archdioceses, and dioceses.

## Spatial Grid & Noisy Edges

`MapOverlayManager` builds a spatial grid mapping each pixel to a cell ID via nearest-center Voronoi assignment. Shared Voronoi edges are then replaced with procedurally noisy edges using Perlin noise, creating medieval-style rough boundaries.

**NoisyEdgeStyle** parameters: `SampleSpacingPx` (2.5), `MaxSamples` (96), `Roughness` (0.58), `AmplitudePerResolution` (0.9), `AmplitudeCap` (8.0), `BandPaddingPx` (1.5).

## Road/Path Rendering

- Road mask texture (`_RoadMaskTex`, R8) generated at border resolution via thick-line rasterization
- Dashed paths for footpaths, solid for roads
- Rendered in market mode only: black routes blended over market color with `_PathOpacity`
- Dynamic — regenerates when `RoadState` changes (hash-based dirty check)

## Selection & Hover

Mode-aware GPU selection/hover with dimming and desaturation of non-selected areas.

**Selection uniforms** (normalized IDs, -1 = none): `_SelectedRealmId`, `_SelectedProvinceId`, `_SelectedCountyId`, `_SelectedMarketId`

**Hover uniforms:** `_HoveredRealmId`, `_HoveredProvinceId`, `_HoveredCountyId`, `_HoveredMarketId`

**Effect uniforms:** `_SelectionDimming`, `_SelectionDesaturation`, `_HoverIntensity`

## Map Modes

| Shader `_MapMode` | Name | Description |
| --- | --- | --- |
| 0 | Height gradient | Elevation-only coloring, no overlay |
| 1–3 | Political | Realm / province / county with drill-down borders |
| 4 | Market | Market zones with road overlay |
| 6 | Biomes | Terrain only, no borders |
| 7 | Channel inspector | Debug channel visualization |
| 8–9 | Transport heatmaps | Transport cost / market access |
| 10–12 | Religion | Archdiocese / diocese / parish with drill-down borders |

## Metal Sampler Budget

Strict 16-fragment-sampler limit on Metal. All border distance textures share a single sampler via aliasing (`sampler_ProvinceBorderDistTex = sampler_RealmBorderDistTex`, etc.).
